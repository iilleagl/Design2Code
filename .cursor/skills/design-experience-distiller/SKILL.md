# 设计经验提炼

从审查报告中提取失败模式，精炼为简洁的经验规则，保存到 `memory/` 目录供后续设计任务参考。

## 输入来源

| 来源 | 内容 |
|------|------|
| `pen-design-component-auditor` | 组件未使用、应用组件却手工搭建等问题 |
| `pen-layout-auditor` | 重叠、超出、间距混乱、缺少交互控件等问题 |
| 设计过程中的错误 | batch_design 报错、ref 失败、copy_components 问题等 |
| 用户反馈 | 对设计结果的手动修正或批评 |

## 工作流程

### 1. 收集审查报告

从当前对话或指定文件中提取所有失败信息。归类为三大类：

| 类别 | 典型问题 |
|------|----------|
| **组件复用** | 复制了没用、应该 ref 却手搭、复制了展示区而非组件 |
| **布局质量** | 重叠、超出、间距为 0、对齐错位、尺寸不合理 |
| **交互完整性** | 缺按钮、缺关闭、缺分页、缺搜索入口 |

### 2. 提炼经验规则

对每个失败问题，提炼为一条**经验规则**，格式严格遵循：

```
## [编号] [简短标题]

**场景**：[什么情况下会出现这个问题]
**问题**：[具体表现]
**规则**：[应该怎么做的一句话总结]
```

**提炼原则**：
- 一条规则只说一件事
- 标题 ≤ 15 字
- 场景、问题、规则各 1-2 句话
- 去掉具体的节点 ID、文件路径等临时信息，保留**通用模式**
- 相同根因的多个问题合并为一条规则

### 3. 去重与合并

读取 `memory/` 目录中已有的经验文件，检查新规则是否与已有规则重复：
- **完全重复**：跳过
- **类似但有新细节**：更新已有规则，补充新的场景或问题描述
- **全新**：追加到对应分类文件

### 4. 写入 memory

经验按分类保存到 `memory/` 目录：

```
memory/
├── component-reuse.md      # 组件复用类经验
├── layout-quality.md       # 布局质量类经验
└── interaction-pattern.md   # 交互完整性类经验
```

每个文件结构：

```markdown
# [分类名] 设计经验

> 从历次设计审查中提炼的经验规则，供设计 agent 参考。

---

## 001 [标题]

**场景**：...
**问题**：...
**规则**：...

---

## 002 [标题]

...
```

编号在文件内连续递增。

## 经验规则示例

### 组件复用类

```markdown
## 001 复制组件必须 ref 使用

**场景**：从组件库 copy_components 复制了导航栏/表单项等组件到设计稿
**问题**：复制后未通过 ref 引用，而是手动搭建了相同区域，导致组件闲置
**规则**：每个 copy_components 复制的组件，必须在对应区域通过 I({type:"ref"}) 实例化使用
```

```markdown
## 002 只复制组件根节点

**场景**：需要使用组件库中的表单项/按钮等组件
**问题**：错误地复制了整个展示区（含多个变体），而非单个组件根节点
**规则**：copy_components 的 ID 必须是 reusable:true 的单个组件，不是包含多个变体的容器
```

### 布局质量类

```markdown
## 001 同级组件不得重叠

**场景**：在同一个 frame 内放置多个组件
**问题**：两个按钮/输入框的布局矩形部分重叠，点击区域冲突
**规则**：同一父容器内的可见子节点之间必须保持 ≥ 8px 间距，使用 layout+gap 自动排列
```

```markdown
## 002 使用 layout 避免手动定位

**场景**：在容器内排列多个子元素
**问题**：手动设置 x/y 导致对齐错位、间距不一致
**规则**：优先使用 frame 的 layout:"vertical"/"horizontal" + gap，而非手动 x/y 定位
```

### 交互完整性类

```markdown
## 001 表单必须有提交按钮

**场景**：设计包含多个输入项的表单区域
**问题**：表单只有输入框没有提交/保存按钮，用户无法完成操作
**规则**：表单底部必须包含主按钮（提交/保存），建议同时提供次要按钮（取消）
```

## 使用已有经验

当 `memory/` 目录中已有经验文件时，设计 agent 可在设计前读取相关经验：

```
Read(memory/component-reuse.md)
Read(memory/layout-quality.md)
Read(memory/interaction-pattern.md)
```

在设计循环中对照经验规则自检，避免重复犯错。
