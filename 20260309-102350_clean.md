## 页面设计需求提示词（Dashboard 首页）

请在 Pencil 中基于 `devUI2` 设计体系，创建一个 Web 管理后台的「项目工作台 / Dashboard 首页」界面，整体为浅色背景、高留白、卡片化风格，宽屏（约 1440–1600px）布局。

### 1. 整体布局
- **页面结构**：采用三栏布局。
  - **左侧固定纵向导航栏**（窄栏，约 64–72px 宽），包含 Logo / 产品名、折叠菜单按钮和若干导航入口图标。
  - **中间主内容区域**（自适应宽度），承载欢迎标题、最近访问、项目列表等核心内容。
  - **右侧信息侧栏**（约 320–360px 宽），用于展示公告、活动和帮助文档入口卡片。
- **背景与分区**：整页浅灰/近白背景（如 `#F5F6FA`），顶部导航区和右侧卡片有轻微投影和圆角。主内容区域整体放在一块大白色内容面板上（可用 devUI 的内容容器/白色卡片组件），四周有较大外边距。
- **栅格与间距**：参考 devUI2 的栅格系统，主内容区左右保持统一边距（例如 24–32px），卡片之间保持统一间距（16–24px），组件圆角和投影与 `devUI2` 默认卡片风格一致。

### 2. 顶部导航栏
- **位置与形式**：页面最上方贯穿整页宽度的顶部导航条，高度约 56–64px，白色背景，底部有极细分割线或轻微阴影。
- **左侧内容**：
  - 左上角放置品牌区：包含品牌 Logo 图标和「华为云」（可使用 devUI 的 Logo/图标组件 + 文本组合）。
  - 紧邻品牌位置放置全局导航 Tab 或文字菜单，名称分别为「控制台」「华北-北京四」「工作台」「效能洞察」「服务」「华为开源镜像站」，使用 devUI2 的顶部导航/Tab 组件风格。
- **右侧内容**：
  - 通知图标，必填图标，语言切换「EN」图标，使用devUI的顶部导航栏-右侧图标组件，均使用默认状态。
  - 用户头像圆形图标，右侧展示用户昵称缩写（「HZ」），使用 devUI 的头像信息组件-默认状态。
  - 右上角整体对齐，元素间距统一。

### 3. 左侧纵向导航栏
- **组件**：整体使用 devUI2 组件库中 `frame=业务组件（更多卡片）` 下的 **左侧选择菜单卡片** 组件，高度撑满视口，背景为浅灰/白色，与主内容有明显分割。
- **内容结构**（与组件库左侧选择菜单卡片结构保持一致，仅替换为当前页面菜单项）：
  - **顶部**：汉堡菜单图标按钮，用于展开/收起菜单。
  - **中部**：一列竖直排列的图标菜单项「我的关注」「默认项目」「API服务创建的项目」「已归档」，复用该组件的菜单项样式与选中态（背景浅色块 + 主色图标），其中「默认项目」为悬浮状态，其他为默认状态。

### 4. 主内容区域（中间）
#### 4.1 顶部 Banner 欢迎卡片（含最近访问入口）
- **位置**：主内容区域顶部偏左，紧邻右侧信息侧栏左侧，对齐主内容白色内容面板的左右边距。
- **组件**：整体使用 devUI2 组件库中 `frame=业务组件（更多卡片）` 下的 **Banner 组件** 作为大卡片容器，白底、圆角、轻微投影。
- **内容结构**（与组件库 Banner 结构保持一致）：
  - **上方标题区**：大号标题文本「Hello, Jingwen」，使用 devUI 标题文字风格（H1/H2），左对齐，黑色或深灰色。
  - **次级标题行**：标题下方一行小号文字「最近访问」，作为该 Banner 的分组标题/说明文字，字号和颜色略弱于主标题。
  - **最近访问 Banner 卡片组**：在「最近访问」文字下方，横向并排放置 3 张完整的 Banner 子卡片，直接复用 `devUI2` 中 `frame=业务组件（更多卡片）` 的 Banner 子卡片结构（图标/Logo + 项目名称）。
    - 项目名称分别为「Build持续演进个人项目规划说明」「DigitalCity智慧城市」「ROMA持续演进个人项目」
    - 三张卡片宽度和间距一致，整体与主 Banner 的左右边距对齐。
    - 卡片视觉样式（背景色、圆角、投影、文字层级）与组件库中同类型 Banner 卡片保持一致，仅替换为当前页面的项目名称与文案。

#### 4.2 所有项目区
- **整体容器**：欢迎区和最近访问下方是一个大块白色卡片容器，标题为「所有项目」，占据中间区域的主要高度。
- **标题行**：
  - 左侧主标题文字「所有项目」。

- **过滤与视图控制行**（在 Tab 下方）：
  - 标题下有一个 Tabs 级别的子导航，包含项目类型标签「全部类型」「项目群」「Scrum」「看板」，使用 devUI 的 页签Tabs 组件，其中「全部类型」为悬浮态，其他为more吗。
  - Tab 右侧放置一个主色按钮「+ 新建」，使用 devUI 的主按钮（实心色，带图标）。
  - 按钮右侧放置一个 面包屑组件「所有项目」，使用 devUI 的基础面包屑组件（active状态）。
  - 面包屑右侧放置一个搜索框，灰色边框圆角输入框，右侧带放大镜图标，提示文案为「请输入关键字搜索」，使用devUI的分类搜索组件（默认状态）。
  
- **基本卡片网格**：
  - 在过滤行下方展示 2 行 x 3 列的项目卡片网格（总共 5 张卡片，第二行末位留空），卡片宽度统一（`fill_container`），左右两列对齐，中间列居中。
  - 每张卡片通过 `ref` 引用组件库中 `frame=基本卡片` 下的 **基本卡片** 组件（默认状态）。**ref 必须同时设置显式的 `width` 和 `height`**（如 `width: "fill_container", height: 164`），否则布局引擎可能无法计算尺寸导致渲染失败。
  - 组件原始结构保持不变（不增删层级），仅通过 `descendants` 覆盖以下文本内容：
    - **标题区**（组件内 `.有图标标题` 行）：保留组件自带的项目首字母图标和收藏星标，通过 descendants 将标题文本节点替换为对应的项目名称（如「MO 安全编程规范」「DigitalCity 智慧城市」「Build 持续演进」等）。
    - **描述区**（组件内灰色描述文本节点）：替换为项目附加信息，如「创建者：abc_abcdefghijklmn\n项目 ID：a2b3c4d5」。
    - **底部信息行**（组件内三组「图标+文字」信息展示节点）：保留组件自带图标，将文本分别替换为项目相关的状态信息（如「系统内置」「232」「预览」，可按需改为其他文案）。
  - 卡片底部保持统一的内边距和行高，确保多张卡片在网格中高度一致。
  - **技术要点**：将 5 张卡片按 2 行排列（第 1 行 3 张，第 2 行 2 张 + 1 个空位 frame），每行使用一个 `layout: "horizontal"` 的 frame 容器，行间 `gap: 16`，行内卡片间 `gap: 16`。

### 5. 右侧信息侧栏
右侧栏整体由三块垂直堆叠的白色信息卡片组成，边距和圆角与中间区域卡片统一。

#### 5.1 公告卡片
- **组件来源**：直接复用 组件库 中 `frame=帮助文档卡片` 下的 **公告卡片** 组件（默认状态），保持原有卡片宽度、圆角和阴影。
- **位置**：右侧栏顶部卡片区域。

#### 5.2 活动卡片
- **组件来源**：直接复用 组件库 中 `frame=帮助文档卡片` 下的 **活动卡片** 组件（默认状态）。
- **位置**：公告卡片下方的中间卡片区域。
- **内容结构**：  
  - 右侧保留组件自带的插画/图片区域（蓝紫渐变、系统面板或云元素风格），左侧为文字内容。  
  - 标题为「活动」，下方为活动名称「频繁 重新登录？」和一句简介说明「点击查看如何增加登录保持时长」。  
  - 底部保留组件中的主色按钮。

#### 5.3 帮助文档卡片
- **组件来源**：直接复用 组件库 中 `frame=帮助文档卡片` 下的 **Banner 帮助文档** 组件（默认状态）。
- **位置**：右侧栏最下方卡片区域。
- **内容结构**：  
  - 顶部标题为「帮助文档」，右侧保留组件内的「所有文档」按钮。  
  - 下方为组件自带的图文入口网格（3 行 x 4 列-3个网格=9个网格）——每个入口为线性图标 + 一行短标题（「产品介绍」「快速入门」「用户指南」「最佳实践」「常见问题」「计费说明」「通用参考」「用户论坛」「版本历程」）。  
  - 使用组件原有的图文栅格/快捷入口布局，保持图标对齐、间距和视觉节奏一致。

### 6. 视觉风格与交互状态
- **色彩**：
  - 主色调为冷色系蓝紫（接近华为云 DevCloud 品牌色），用于按钮、选中 Tab、Tag 和图标高亮。
  - 背景为浅灰/近白，卡片为纯白。
  - 文本主标题使用深灰/近黑，中灰用于描述文本，浅灰用于提示/次要信息。
- **圆角与投影**：
  - 卡片、搜索框、按钮等圆角半径统一（参考 devUI2 规范，如 4px 或 6px）。
  - 卡片采用轻微阴影或边框分割，让层级清晰但不过度突出。
- **组件状态**：
  - Tab、按钮、Tag、图标等都体现 hover/selected 状态（例如 Tab 选中时使用主色下划线，按钮 hover 色略深，左侧导航选中项有浅色背景块）。
  - 卡片整体在 hover 时可略微提升阴影或轻微放大。

### 7. 组件使用要求（基于 devUI2）
- 尽量使用 `devUI2` 设计系统中已有的组件组合页面：
  - 顶部导航组件（含 Logo + 全局菜单 + 用户区）。
  - 左侧纵向导航栏使用 `frame=业务组件（更多卡片）` 下的 **左侧选择菜单卡片** 组件（图标型菜单，支持选中态）。
  - 标题文字、说明文字的标准 Typography。
  - 卡片组件（用于最近访问、项目卡片、公告卡片、活动卡片、帮助文档卡片）。
  - Tabs 组件（项目类型切换）。
  - 按钮组件（主按钮、次级按钮、文本按钮）。
  - Tag/Badge 组件（项目状态、标签）。
  - 输入框/搜索框组件，下拉选择器组件，图标按钮组件。
  - 图文快捷入口组件或图标 + 文本的宫格布局组件。
- 布局上参考 devUI2 的栅格和间距体系，使整个页面层级清晰，左右对齐，卡片分布与示例图基本一致。

### 8. 目标
- 生成的页面应在不依赖具体文案一字不差的前提下，**整体布局、结构分区、组件类型、数量和相对位置尽量还原截图中的 Dashboard 首页**，让设计师或自动生成工具可以按上述描述在 Pencil 中搭建出几乎相同的页面。
